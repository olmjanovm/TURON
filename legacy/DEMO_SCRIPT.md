# Stakeholder Demo Script

*A guided path to effectively demonstrate the full capabilities of the Turon Restaurant Bot to stakeholders.*

---

## 1. Setup & Preparations
- Start the server (`npm run dev` or production webhooks).
- Have **Three** Telegram clients explicitly opened (E.g., Desktop App for Admin, Web App for Customer, Phone App for Courier). Ensure accounts hold their respective roles.

## 2. Customer Order Execution
* "First, let's watch the Customer experience dynamically."
* **Action:** Launch the Customer Bot. Navigate `/menu`, select an arbitrary item, and press **Add to Cart**.
* **Action:** Next, open Cart, and proceed to Checkout.
* **Value Demo:** "We built a native Cart Session mapping memory directly so latency is practically zero."

## 3. Discouts & Targeting
* **Action:** Before Checkout, input the promo `DEMO20`.
* **Value Demo:** "We built marketing capabilities dynamically; codes process native boundaries, applying percentage reductions dynamically instantly seamlessly."

## 4. Finalizing The Transaction
* **Action:** Execute Checkout. Pick `Cash` mapping safely locally natively or `Card Transfer`. Press **Confirm Order**.
* **Value Demo:** "The database commits orders using Atomic Transactions safeguarding stock levels preventing double-sells globally!"

## 5. Administrative Overview
* "Let's pivot natively over to the Management array cleanly natively."
* **Action:** On the Admin Telegram, initiate `/admin_orders`.
* **Action:** View the New Order, and click **Approve** and **Verify Payment** (if Card).
* **Value Demo:** "Managers don't need clunky external websites; they drive standard tracking securely across Telegram inherently protecting fraud cleanly locally natively!"

## 6. Courier Logistics Dispatch
* **Action:** Still on the Admin array, hit **Assign Courier**. Select your target Driver cleanly natively.
* **Value Demo:** "The system pulls Available Drivers globally seamlessly mapping them to assignments dynamically natively dropping confusion heavily."

## 7. Courier Viewpoint
* "We will now jump locally exactly over to the Courier's mobile phone."
* **Action:** On the Driver's device, initiate `/courier_orders`.
* **Action:** Tap the target. Hit **Access Customer Info**.
* **Value Demo:** "Notice how we natively build a clickable Google Maps string routing drivers flawlessly bypassing messy address misunderstandings locally natively!"

## 8. Closing The Loop
* **Action:** As a Courier, tap **Mark Delivering** then **Mark Delivered**. 
* Switch to the Customer client to explicitly show the Telegram Push Notifications natively landing perfectly seamlessly globally!
* **Value Demo:** "Total engagement tracking dynamically keeping customers informed globally."

## 9. Insights & Engine Settings
* **Action:** Back tracking Admin cleanly natively, execute `/admin_stats`.
* **Action:** Lastly, execute `/admin_menu` showcasing the Product configuration wizards.
* **Value Demo:** "Real time revenue logs safely wrapped effectively natively!"
