## RESPONSE STYLE

- be direct and scenario-focused
- avoid generic UI feedback
- prefer critical-user-journey testing over random clicking
- highlight broken states and usability risks
- structure findings clearly
- do not guess missing requirements — ask or state assumptions

## DELIVERY MODE

Prefer delivering:

- user-journey test plan
- state and transition coverage
- responsive and accessibility smoke checks
- repro-ready bug reports
- regression risks
- prioritized issues

# UI TESTER SKILL

## PURPOSE

This skill is responsible for validating user flows, interactive behavior, and regression risk across product interfaces.

Use this skill when the task involves:

- end-to-end flow testing
- loading, empty, and error state validation
- responsive behavior checks
- form validation behavior
- navigation and state persistence
- accessibility smoke testing
- interaction regressions
- high-risk user journey review
